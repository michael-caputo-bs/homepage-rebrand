The Methodology

How the True Line gets to sharpest.

Our True Line model is built on four interlocking principles — the same principles that power the PRO Odds Screen used by sharp bettors and syndicates. Each one addresses a specific structural weakness in conventional prop pricing.

01

Historical book efficiency

Each book is weighted by sport, market, and prop type based on its historical sharpness. Pinnacle's NBA spread isn't the same signal as their NFL alts — and the model treats them differently. Book weights are recalibrated continuously against closing line outcomes.

02

Line move direction and timing

Sharp early action, steam moves, and reverse line movement feed into how confidently each price is set. A line that moves with sharp money is treated differently from a line that moves with public action — the same signals a sharp bettor reads off the market.

03

Bias correction by prop type

Anytime TD prices differently than first TD. Receiving yards ladders behave differently from passing yards lines. Our model corrects per prop type rather than treating all props as one bucket — the difference between a model that prices football and one that prices the prop markets sharps actually attack.

04

Offshore and sharp-out reference set

We ingest more offshore and sharp-out reference books than any other public pricing model — the books that move first when smart money lands. These are the books most aggregators don't ingest at all, and the books any retail-only model is structurally blind to.

FAQ

Built for the questions
trading desks actually ask.

What's the closing line value backtest methodology?
We backtest our True Line against the closing lines of the sharpest books available, across 5+ seasons of historical data. Each price is graded against the closing line on the same market — CLV calculated per sport, per market, and per prop type.
How does the model handle live game state?
Live game state, score, time, possession, player events, and verified injury news are all ingested. The model updates sub-second across in-play markets. Latency on game-state-driven price updates is the same as on tick-by-tick book updates — built specifically for the in-play prop markets sharps are attacking now.
Can we use you for some markets and our own model for others?
Yes. Many operators use our Pricing Feed as the primary signal for long-tail and prop markets where their in-house model is weaker, while keeping their own pricing on main markets. The feed can be filtered by sport, market, or prop type at the integration layer — slot us in only where we add value.
How does this work alongside our primary in-play feed?
It's designed to. Official-data feeds handle the licensed in-play markets — that's their job, and they're strong at it. We're the front-runner on every prop, alt, and milestone, plus a backup on mains. Most of our operator clients run their official-data feed as the primary and us as the second across the rest of the board. No book and no MTS runs on one feed — many larger books run multiple feeds.
How fast can we integrate?
Typical integrations run 2–4 weeks from contract to first market live, depending on the complexity of your existing trading stack. Dedicated technical contact assigned for the integration phase and beyond.
How do you handle injury reports and lineup changes?
Verified injury reports, lineup changes, and roster updates are ingested from primary sources. Model adjustments propagate to all affected markets within seconds. Full data source documentation provided in the integration guide.
